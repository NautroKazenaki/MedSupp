# Домашняя аптечка

PWA для общего учёта домашних лекарств: две учётные записи, сроки годности и количество таблеток. Фронтенд разворачивается на Vercel, а каждое лекарство хранится отдельным JSON-файлом в Vercel Blob.

## Локальный запуск

```bash
npm install
cp .env.example .env.local
npm run dev
```

Для работы API локально создайте `.env.local` по образцу `.env.example`.

## Настройка Vercel

1. Создайте проект в Vercel и подключите Git-репозиторий.
2. В разделе **Storage** создайте **Blob Store** и подключите его к проекту. Vercel добавит `BLOB_READ_WRITE_TOKEN`.
3. В **Settings → Environment Variables** добавьте `SESSION_SECRET`, `USER_1_NAME`, `USER_2_NAME`, `USER_1_PASSWORD_HASH` и `USER_2_PASSWORD_HASH`.
4. Создайте хеш для каждого пароля локально:

   ```bash
   node -e "import('bcryptjs').then(({hash}) => hash('ваш-пароль', 12).then(console.log))"
   ```

   В Vercel следует добавлять получившиеся хеши, а не сами пароли. Значения из `.env.local` не коммитятся.
5. Сделайте deploy. Vercel автоматически соберёт Vite-фронтенд и развернёт обработчики из папки `api/`.

## Устройство данных

Каждая запись — это JSON-файл в Blob с именем `medicines/<uuid>.json`. Это позволяет Сергею и Дарье добавлять и редактировать разные лекарства без общей базы данных.

Доступ к приложению защищён именем пользователя, bcrypt-хешем пароля и подписанной `HttpOnly`-cookie. Cookie не доступна JavaScript и не хранит пароль.

## Важное ограничение Vercel Blob

Vercel Blob — файловое, а не транзакционное хранилище. При одновременном редактировании одной и той же записи приложение покажет конфликт и попросит обновить список. У Blob-файлов также есть публичные URL: они не выдаются интерфейсом до входа, но Blob не подходит для конфиденциальных медицинских данных. Для усиленной приватности и полноценного параллельного редактирования используйте базу данных с авторизацией.

## PWA

После первого открытия сайт можно установить как приложение из меню браузера: «Добавить на главный экран» на телефоне или «Установить приложение» на компьютере. Интерфейс работает из кэша офлайн; синхронизация лекарств требует сети.

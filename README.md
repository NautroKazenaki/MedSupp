# Домашняя аптечка

PWA для учёта домашних лекарств: несколько учётных записей, общие или личные аптечки (household), сроки годности и количество таблеток. Фронтенд разворачивается на Vercel, а каждое лекарство хранится отдельным JSON-файлом в Vercel Blob.

## Локальный запуск

```bash
npm install
cp .env.example .env.local
npm run dev
```

Заполните в `.env.local` пароли и при необходимости имена/household. Для работы API локально нужен доступ к Blob (см. ниже).

## Настройка Vercel

1. Создайте проект в Vercel и подключите Git-репозиторий.
2. В разделе **Storage** создайте **Blob Store** и подключите его к проекту. Vercel добавит `BLOB_STORE_ID` (и при наличии — `BLOB_READ_WRITE_TOKEN`).
3. В **Settings → Environment Variables** добавьте:
   - `SESSION_SECRET`
   - для каждого пользователя: `USER_N_NAME`, `USER_N_PASSWORD_HASH`, `USER_N_HOUSEHOLD` (N = 1…20)
4. Несмотря на суффикс `_HASH`, в этой упрощённой версии в `USER_N_PASSWORD_HASH` кладётся обычный пароль. Значения из `.env.local` не коммитятся.
5. Сделайте deploy. Vercel автоматически соберёт Vite-фронтенд и развернёт обработчики из папки `api/`.

### Пример: семья и отдельные аптечки

| Пользователь | HOUSEHOLD | Что видит |
|--------------|-----------|-----------|
| Сергей, Дарья | `family` | одна общая аптечка |
| Катя | `katya` | только своё |
| Полина | `polina` | только своё |
| Маргарита | `margarita` | только своё |

`HOUSEHOLD` — латиница/цифры/`_`/`-` (например `family`, `katya`). Одинаковый household у двух логинов = общая аптечка.

## Устройство данных

Каждая запись — JSON-файл `medicines/<household>/<uuid>.json`. API читает и пишет только каталог household из сессии пользователя.

Доступ защищён именем, паролем из переменных Vercel и подписанной `HttpOnly`-cookie. Cookie не доступна JavaScript и не хранит пароль. В сессии также хранится `household`.

### Миграция старых данных (один раз)

Vercel Blob разрешает работу с production-данными только из Vercel Production. Поэтому миграция выполняется защищённым API на Vercel, а локальная команда лишь запускает его. По умолчанию операция только копирует файлы — старые пути не удаляются. Существующие файлы в `medicines/family/` не перезаписываются.

1. В Vercel → **Settings → Environment Variables** добавьте:

```
MIGRATE_SECRET=<длинная_случайная_строка>
```

2. Задеплойте этот код.

3. В локальном `.env.local` добавьте:

```bash
MIGRATE_URL=https://<ваш-проект>.vercel.app
MIGRATE_SECRET=<то же значение, что в Vercel>
```

4. Запустите проверку, затем копирование:

```bash
npm run migrate:household:dry
npm run migrate:household
```

5. Выйдите и войдите как Сергей/Дарья — аптечка должна быть на месте.

6. Только после проверки удалите старые плоские файлы. Эта команда не копирует данные заново: она сначала убеждается, что у каждого старого файла есть копия в `medicines/family/`, затем удаляет только старый путь.

```bash
npm run migrate:household:cleanup
```

7. Удалите `MIGRATE_SECRET` из Environment Variables Vercel и из `.env.local`.

После деплоя все пользователи должны **выйти и войти снова** (формат cookie изменился).

## Важное ограничение Vercel Blob

Vercel Blob — файловое, а не транзакционное хранилище. При одновременном редактировании одной и той же записи приложение покажет конфликт и попросит обновить список. У Blob-файлов также есть публичные URL: они не выдаются интерфейсом до входа, но Blob не подходит для конфиденциальных медицинских данных. Для усиленной приватности и полноценного параллельного редактирования используйте базу данных с авторизацией.

## PWA

После первого открытия сайт можно установить как приложение из меню браузера: «Добавить на главный экран» на телефоне или «Установить приложение» на компьютере. Интерфейс работает из кэша офлайн; синхронизация лекарств требует сети.
